

# 現状の問題点と不足している要素

## 1. 致命的なバグ: Base64エンコーディングエラー (最優先)

エッジファンクションのログで **全リクエストが400エラー** になっています。原因は51行目のパディング処理:

```
const paddedBase64 = base64Data + "=".repeat((4 - (base64Data.length % 4)) % 4);
```

`canvas.toDataURL()` は既に正しいbase64を出力するため、このパディング追加が逆にデータを破壊しています。パディング処理を削除し、元のbase64をそのまま使う必要があります。

## 2. 不足している機能

| 項目 | 内容 |
|------|------|
| **エラーUI表示** | AI分析失敗時にユーザーへのフィードバックがない。`useAIGuide`の`error`状態がGuidePage上で表示されていない |
| **リトライ制御** | 3秒間隔のポーリングでエラーが連続すると大量リクエストが飛ぶ。エラー時のバックオフ/リトライ上限が必要 |
| **ローディング初期状態** | カメラ起動〜初回AI応答までの間、ユーザーに「分析中...」のフィードバックがない |

## 実装プラン

### タスク1: Edge Function のbase64修正
- `supabase/functions/analyze-frame/index.ts` のパディング処理を削除
- base64データをそのまま `data:image/jpeg;base64,{raw}` として送信

### タスク2: エラーハンドリング改善
- `useAIGuide.ts`: エラー連続時にポーリング間隔を延長するバックオフ追加（3秒→6秒→12秒、最大30秒）
- 3回連続失敗でポーリング一時停止

### タスク3: GuidePage UIフィードバック
- AI分析エラー時のトースト/HUD表示を追加
- 初回分析待ちの「接続中...」表示を追加
